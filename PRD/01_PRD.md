## 1. 개요

### 1.1 제품 한 줄 설명
딸기 손질 전용 가정용 스마트 로봇 **베리봇(BerryBot)** 구독 서비스를 소개하는 원페이지 정적 랜딩 사이트.

### 1.2 목적 및 배포
| 항목 | 내용 |
|------|------|
| 목적 | 과제 제출용 랜딩 페이지 |
| 배포 플랫폼 | GitHub Pages |
| URL 형태 | `https://[username].github.io/berrybot-landing/` |
| 빌드 도구 | 없음 (단일 HTML 직접 배포) |

### 1.3 해결하는 문제
> "딸기 손질이 번거롭다 — 세척, 꼭지 제거, 다듬기까지 시간이 걸린다"

베리봇은 이 번거로운 과정을 **가정용 구독 로봇**으로 자동화한다.
배송·신선도 보증 서비스가 아닌 **가전 구독 서비스**임을 랜딩 페이지에서 명확히 전달해야 한다.

---

## 2. 타깃 사용자

- 딸기·베리를 자주 먹는 가정 (아이가 있는 가정 포함)
- 손질 시간을 절약하고 싶은 1~4인 가구
- 스마트 가전에 관심 있는 2030~40대

## 3. 페이지 구성 요구사항

### 3.1 섹션 목록

| 순서 | 섹션 ID | 섹션명 | 상태 | 우선순위 |
|------|---------|--------|------|----------|
| 1 | `#hero` | Hero (서비스 소개 + CTA) | ✅ 완성 | P0 |
| 2 | `#comic` | 4컷 만화 (손질 과정) | ✅ 완성 | P0 |
| 3 | `#plans` | 구독 플랜 비교 | ✅ 완성 | P0 |
| 4 | `#features` | 특장점 카드 그리드 | ✅ 완성 | P0 |
| 5 | `#spec` | 로봇 사양 표 | 🔧 추가 필요 | P1 |
| 6 | `#footer` | 푸터 (사업자 정보) | ✅ 완성 | P0 |

### 3.2 섹션별 콘텐츠 요구사항

#### #hero
- 뱃지: "🤖 딸기 손질 전용 로봇"
- 메인 타이틀 (2줄, `<accent>` 강조)
- 서브 카피 (한 줄 설명)
- 긴급성 문구 (첫 달 50% 할인)
- CTA 버튼 2개: `btn-primary` (구독 시작) + `btn-ghost` (소개 보기)
- 히어로 로봇 이미지 (`panel4.svg` 재사용, float 애니메이션)

#### #comic
- 섹션 태그 + 제목 + 설명
- 4개 패널 (`panel1~4.svg`), 2×2 그리드 → 데스크톱 4열
- 각 패널에 `data-reveal` 스크롤 애니메이션

**패널별 장면 요구사항**

| 패널 | 파일 | 장면 요약 | 핵심 시각 요소 |
|------|------|-----------|----------------|
| 1컷 | `panel1.svg` | 딸기·블루베리를 바구니에서 투입구에 넣는 장면 | 둥근 바구니(딸기 5개+블루베리 4개), 오른팔 뻗은 베리봇, 크림 배경 |
| 2컷 | `panel2.svg` | 싱크대에서 물로 과일을 세척하는 장면 | 수도꼭지, 물 채워진 싱크(전경 레이어), 딸기 3개+블루베리 4개, 물방울 튀기 효과, 파란 배경 |
| 3컷 | `panel3.svg` | AI 돋보기로 꼭지 위치 검수·정밀 제거 장면 | 황금 검수 램프, 작업대, 돋보기(원+핸들 일체형), 떨어진 꼭지, 노란 배경 |
| 4컷 | `panel4.svg` | 손질 완료 과일이 그릇에 담겨 나오는 완성 장면 | 딸기·블루베리 혼합 그릇(전경 레이어), 컨페티 배경, 베리봇 만세 포즈, 분홍 배경 |

**일러스트 공통 규칙**
- SVG viewBox: `0 0 400 500`, 폰트: `'Apple SD Gothic Neo', 'Noto Sans KR', Arial`
- 베리봇 캐릭터: 딸기 모양 몸통(레드 그라디언트 `#FF7090→#FF3355→#BB0030`), 잎 왕관, 회색 다리·발
- 다리: `y="368"` (몸통 하단에 붙임), 발 중심: `cy="396"`
- z-order 원칙: 배경 장식 → 배경 소품(싱크대·작업대·바구니) → 베리봇 → **전경 소품(물·그릇·돋보기)** → 말풍선·패널번호
- 캡션 영역: `y=420` 반투명 흰 배경 + 패널별 강조색 3px 상단 라인 + 제목(20px bold) + 부제(12px)
- 패널 번호: 좌상단 빨간 원(`r=20`, `fill=#FF3355`) + 흰 숫자

#### #plans
- 라이트 🍓 / 홈 🫐 (인기) / 패밀리 🍒 3개 카드
- 홈 플랜: `plan-best` 강조 + 뱃지 + 소셜 프루프
- 모든 플랜: 첫 달 50% 할인 주석

#### #features
- 6개 카드: 초음파 세척 / 꼭지 제거 / 신선도 식별 / 커팅 / 재손질 / 쉐이크 모드
- 3열 그리드, 각 카드 `data-reveal`

#### #spec (신규 추가)
- 제목: "베리봇 로봇 사양"
- 4개 부위 설명 표:

| 부위 | 명칭 | 역할 |
|------|------|------|
| 상단 입구 | 호퍼(Hopper) | 딸기 한꺼번에 투입 |
| 중앙부 | 클린 존(Clean Zone) | 미세 버블 + 초음파 3단 세척 |
| 작업대 | 트리밍 데크(Trimming Deck) | AI 카메라 꼭지 감지 + 소프트 그리퍼 제거 |
| 하단 출구 | 서빙 트레이(Serving Tray) | 완료된 딸기 서빙 |

- 추가 스펙: 크기 (10인용 전기밥솥 수준), 소재 (무광 화이트/파스텔 유선형), 투명 돔
- **반응형 규칙**:
  - 데스크톱(1024px↑): 개요 카드 3열 가로 배치 + 테이블 기본 표시
  - 태블릿(768px): 개요 카드 3열 유지, 테이블 셀 `white-space: normal` 적용 (줄바꿈 허용)
  - 모바일(480px↓): 개요 카드 1열 세로 스택, 테이블을 카드형 리스트로 변환 (`thead` 숨기고 각 행을 개별 카드 표시)

#### #footer
- 로고 + 태그라인
- 이용약관 / 개인정보처리방침 / 고객센터 링크
- 사업자 정보 (상호, 대표, 사업자번호, 주소, 통신판매업 신고)
- 고객센터 전화 / 이메일

### 3.3 전역 인터랙션 요구사항

| 컴포넌트 | 요구사항 |
|----------|----------|
| 네비게이션 바 | fixed top, 스크롤 시 blur 배경 |
| 모바일 햄버거 | 768px 이하에서 표시, 링크 클릭 시 자동 닫힘 |
| 스크롤 리빌 | `[data-reveal]` → IntersectionObserver, 형제 간 100ms 딜레이 |
| 플로팅 CTA | 히어로 벗어나면 표시, #plans 이동 |
| 히어로 로봇 | 4s ease-in-out 무한 부유 애니메이션 |
| FOUC 방지 | `<head>` 인라인 `[data-reveal]{opacity:0;transform:translateY(30px)}` |

### 3.4 반응형 동작 규칙

브레이크포인트: 모바일(480px), 태블릿(768px), 데스크톱(1024px)

| 섹션 | 데스크톱 (1024px↑) | 태블릿 (769~1024px) | 모바일 (≤768px) | 소형 모바일 (≤480px) |
|------|---------------------|---------------------|-----------------|----------------------|
| `#hero` | 2열 (텍스트 + 로봇 이미지) | 2열 유지, 로봇 축소 | 1열 세로 스택, 텍스트 중앙 정렬, 로봇 `max-width: 300px` | 동일, 패딩 축소 |
| `#comic` | 4열 그리드 | 2열 그리드 | 2열 그리드, gap 축소 | 1열, `max-width: 320px` |
| `#plans` | 3열 그리드 | 3열 유지 | 1열 스택, `max-width: 380px` | 동일 |
| `#spec` | 개요 3열 + 테이블 기본 | 개요 3열, 테이블 줄바꿈 허용 | 개요 1열 스택, 테이블 카드형 변환 | 동일, 패딩 축소 |
| `#features` | 3열 그리드 | 2열 그리드 | 2열 유지 | 1열, `max-width: 400px` |
| `#footer` | 기본 | 기본 | 사업자 정보 `|` 구분자 줄바꿈 | 동일 |

## 4. 콘텐츠 금지 사항

- 배송·수량·신선도 보증 등 **과일 배송 서비스**로 읽힐 수 있는 표현 사용 금지
- UI 텍스트는 **한국어**로 통일
- 색상 값은 CSS 변수(`--red`, `--green` 등)로만 사용 — hex 직접 입력 금지
