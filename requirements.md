# 요구사항 정의서

## 프로젝트 개요

- **서비스명:** 베리봇 (BerryBot) — 후보: 딸봇 / 프레쉬봇 / 꼭지탈출 / 딸기메이트
- **형태:** 원페이지 웹사이트 (랜딩 페이지)
- **주제:** 딸기 손질 전용 로봇이 구독자의 딸기를 대신 세척·다듬어 배송하는 구독 서비스

---

## 베리봇 — 로봇 정체성

베리봇은 **딸기 손질에 특화된 식품 가공 로봇**이다. 사람 대신 딸기 세척·다듬기·꼭지 제거 작업을 자동으로 수행하며, 구독자에게 바로 먹을 수 있는 상태의 딸기를 정기 배송한다.

| 항목 | 내용 |
|------|------|
| 외형 | 딸기 모양 몸통, 초록 잎 왕관, 노란 씨앗 무늬, 팔·다리·발 장착 |
| 작업 방식 | 딸기 접수 → 자동 세척 → 품질 검수 → 꼭지 제거 → 포장 |
| 표정 변화 | 컷마다 감정 표현 (신남 / 하트눈 / 집중 / 뿌듯) |
| 서비스 포지션 | "로봇이 손질한다"는 점을 핵심 차별화 포인트로 강조 |

---

## 로봇 기능 정의

베리봇(로봇)이 수행하는 핵심 작업 (4컷 만화 스토리 기반):

| 컷 | 동작 | 설명 |
|----|------|------|
| 1컷 | 딸기 받기 | 로봇이 딸기를 전달받는 장면 |
| 2컷 | 세척 | 딸기를 깨끗이 씻는 장면 |
| 3컷 | 다듬기 | 딸기를 정돈/다듬는 장면 |
| 4컷 | 꼭지 제거 | 꼭지를 떼어 완성된 딸기를 건네는 장면 |

## 디자인 요구사항

- **스타일:** 페이지 전체를 관통하는 4컷 만화 하나로 구성 (웹툰/카툰 느낌)
- **테마:** 딸기 컨셉 전반 적용 (빨강/초록/흰색 컬러, 씨앗 패턴, 딸기 아이콘 등)
- **캐릭터:** 딸기 모양 **로봇(베리봇)**이 주인공 — 로봇이라는 정체성을 카피·UI 전반에 명시적으로 노출
  - 히어로 배지, 섹션 설명문, 특장점 카드 등에서 "로봇이 직접 손질"이라는 점을 강조
- **일러스트:** 직접 제작 (외부 에셋 미사용)

## 기술 스택

| 분류 | 선택 | 이유 |
|------|------|------|
| 마크업 | HTML5 | 정적 원페이지, 별도 프레임워크 불필요 |
| 스타일 | CSS3 (순수) | 딸기 테마 커스텀 디자인, 의존성 최소화 |
| 인터랙션 | Vanilla JS | 스크롤 애니메이션, 플랜 탭 전환 등 경량 처리 |
| 일러스트 | SVG (인라인) | 직접 제작 벡터 이미지, 해상도 무관, CSS 애니메이션 연동 가능 |
| 빌드 | 없음 (단일 HTML) | 배포 단순화, 별도 번들러 불필요 |
| 배포 | GitHub Pages / Netlify | 무료 정적 호스팅 |

## 4컷 만화 콘티

> 캐릭터: 베리봇 — 딸기 모양 몸통, 동그란 눈, 팔·다리 달린 귀여운 로봇

---

### 1컷 — "배달 왔어요!"
```
[ 장면 ]
베리봇이 딸기가 가득 담긴 바구니를 두 팔로 받는다.
딸기가 통통 튀어오를 것 같은 느낌.

[ 말풍선 / 효과음 ]
베리봇: "냠냠냠... 오늘도 딸기다!"
효과음: 반짝반짝 ✨

[ 배경 ]
주방 느낌, 딸기 무늬 앞치마 착용
```

---

### 2컷 — "깨끗이 씻어줄게!"

```

### 3컷 — "예쁘게 다듬어볼게!"
